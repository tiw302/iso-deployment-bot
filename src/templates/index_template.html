> project_summary
[overview]
automated iso mirroring system designed to synchronize multiple operating system distributions to google drive storage. the system utilizes github actions for scheduled execution, aria2 for multi-threaded downloads, and rclone for cloud storage synchronization.

[why this exists]
when deploying operating systems across virtualized environments (like proxmox, hyper-v) or physical machines, fetching iso images from slow, geographically distant upstream mirrors can waste hours of time.

this library maintains high-speed, direct google drive mirrors of the most common, secure, and clean system installation media. it guarantees that all deployment files are:
1. hosted on enterprise-grade cloud storage for maximum download bandwidth.
2. verified and synced daily to prevent broken links or stale versions.
3. stored securely, providing a single consolidated inventory.

> frequently_asked_questions
q: how often are the mirrored files synced?
a: all active operating system mirrors are synced daily at 00:00 utc. the index generator runs automatically to update this page with the latest active google drive links.

q: where are these files stored and are they safe?
a: files are stored on high-speed google drive enterprise storage and shared via rclone. all iso and img files are matched against the upstream sha256 checksums to verify integrity and security.

q: how can i request a new operating system distribution or report a broken link?
a: you can submit a request on our github repository's issues page or send a direct message on instagram. check the contact section below for links.
